# Guia do Usuário

## Para quem é este guia

Este guia é para **usuários finais** (colaboradores de campo, revisores e participantes da comunidade) que querem usar o ColaboraPANC no fluxo cotidiano.

Se você é desenvolvedor/mantenedor, use:
- `docs/pt-BR/contribuicao.md`
- `docs/pt-BR/admin.md`

## 1) Valor do sistema na prática

O ColaboraPANC ajuda comunidades e especialistas a:
- mapear observações de PANC com evidência georreferenciada,
- usar IA assistiva para acelerar hipóteses de identificação,
- validar dados com revisão humana,
- enriquecer contexto taxonômico/ecológico,
- acompanhar sinais territoriais/ambientais.

## 2) Perfis de usuário e linguagem por papel

- **Colaborador de campo (usuário final):** captura pontos, imagens e contexto; acompanha status do ponto.
- **Revisor/especialista:** valida ou rejeita registros científicos pendentes.
- **Usuário de comunidade:** consome contexto de mapa, notificações e recursos do ecossistema.

> Importante: saída de IA é assistiva; validação científica final é humana.

## 3) Jornada típica do usuário

1. Entrar na conta ou criar cadastro.
2. Registrar ponto PANC georreferenciado (com imagem/contexto quando possível).
3. Consultar sugestões assistivas de identificação.
4. Acompanhar status de moderação/revisão.
5. Seguir atualizações, alertas e recursos de apoio do ecossistema.

## 4) Fluxo mobile em campo (online + offline)

1. Abrir mapa/captura e coletar observação + imagem.
2. Se online, enviar imediatamente.
3. Se offline, continuar coleta; registros ficam em fila local.
4. Reconectar e sincronizar pendências.
5. Confirmar estado de sincronização na interface antes de encerrar atividade de campo.

Para fluxos avançados (pacotes offline seletivos, auto-detecção, recursos AR), veja:
- `docs/pt-BR/fluxos-mobile-avancados.md`

## 5) Ciclo científico de revisão (o que esperar)

- Pontos podem entrar em fila de revisão.
- Revisor/admin pode validar, rejeitar ou pedir nova revisão.
- Divergência entre IA e revisão final pode ocorrer e é esperada.
- Mudanças de status fazem parte do controle de qualidade, não de falha.

## 6) Boas práticas para contribuições melhores

- Enviar imagens nítidas (folha/flor/fruto/planta inteira quando possível).
- Incluir contexto local (bioma/região/anotações) no relato.
- Evitar registros duplicados do mesmo exemplar/local.
- Não tratar confiança da IA como verdade final.

## 7) Troubleshooting rápido para usuário final

- Sem internet? Continue no fluxo offline e sincronize depois.
- Identificação incerta? Aguarde validação de revisor.
- Atualizações não aparecem? Atualize e verifique estado de sync pendente.
- Problemas de acesso/perfil? Contate a operação do projeto.
